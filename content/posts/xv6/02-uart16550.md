# 给自己加点料——UART16550驱动重写与移植

我准备使用 Rust 重写 UART16550 的驱动。

## 初始化

首先，驱动当然是 `no_std` 的：

```rust
#![no_std]
```

我们注意到，`uart.c` 里定义了一系列常量和宏作为辅助，我们将其迁移到 Rust。

```rust
pub type RegOffset = usize;
pub type RegValue = u8;

// 原 uart.c 中的 UART 寄存器偏移和位定义
pub const UART0: usize = 0x10000000;
pub const RHR: RegOffset = 0;
pub const THR: RegOffset = 0;
pub const IER: RegOffset = 1;
pub const IER_RX_ENABLE: RegValue = 1 << 0;
pub const IER_TX_ENABLE: RegValue = 1 << 1;
pub const FCR: RegOffset = 2;
pub const FCR_FIFO_ENABLE: RegValue = 1 << 0;
pub const FCR_FIFO_CLEAR: RegValue = 3 << 1;
pub const ISR: RegOffset = 2;
pub const LCR: RegOffset = 3;
pub const LCR_EIGHT_BITS: RegValue = 3 << 0;
pub const LCR_BAUD_LATCH: RegValue = 1 << 7;
pub const LSR: RegOffset = 5;
pub const LSR_RX_READY: RegValue = 1 << 0;
pub const LSR_TX_IDLE: RegValue = 1 << 5;

pub const UART_TX_BUF_SIZE: usize = 32;

#[inline]
const fn reg(offset: RegOffset) -> *mut u8 {
    (UART0 + offset) as *mut u8
}

#[inline]
unsafe fn read_reg(offset: RegOffset) -> RegValue {
    unsafe { core::ptr::read_volatile(reg(offset).cast_const()) }
}

#[inline]
unsafe fn write_reg(offset: RegOffset, value: RegValue) {
    unsafe { core::ptr::write_volatile(reg(offset), value) };
}
```

这里使用 `RegOffset` 和 `RegValue` 两个类型别名来增强安全性（编译器会检查传给三个内联函数的参数类型）。

这里仅仅标注 `#[inline]` 而不强制 `#[inline(always)]`，是因为之前在实现 RISC-V 模拟器时，出现了强制内联小函数性能回退的问题——大概与 I-Cache miss 有关。考虑到懒得跑 PGO，仅仅提示编译器进行内联即可。

## UART 驱动分解

首先，驱动提供了 `uartinit()` 函数来进行一系列初始化。

接下来，面向操作系统，驱动提供了 `uartputc`、`uartputc_sync`，为上层提供写入功能。

面向硬件，驱动提供了 `uartstart`，将缓冲区中的数据写入 UART。

`uartgetc` 尝试读取数据。

`uartintr` 处理中断。在这里我们遇到的中断只有两种可能：

1. 有输入数据到达
2. UART 输出完成，可以继续发送

于是，`uartintr` 处理中断：

1. 尽可能循环调用 `uartgetc`，直到没有数据可读，然后将数据发给 `console`
2. 调用 `uartstart`

工作流程大致如下：

- `uartinit` 初始化整个模块
- `uartputc_sync` 不经过缓冲机制，直接将数据写入 UART（如果发送寄存器已满，则等待）
- `uartputc` 将数据写入缓冲区，然后调用 `uartstart` 尝试写入。如果写不完，就等待下一次 `uartstart` 被调用（硬件发中断表示"可以继续写了"）

这是我们"主动"能做的。被动做的是硬件发中断，通知我们可以继续发送/接收数据了。

我们就在 `uartintr` 中尝试接收消息，然后继续调用 `uartstart` 发送消息。

对外的接口只有 `uartputc`、`uartputc_sync`、`uartintr`（在 trap 处被调用）。

需要的外部接口是 `consoleintr`。

需要说明，`uartgetc` 仅仅在 `uartintr` 中被调用。

所以为什么写在 `defs.h` 里了啊！

删了，似乎没影响，交了个小 PR：https://github.com/mit-pdos/xv6-riscv/pull/470

## 设计 UART 模块

我们可以使用面向对象的程序设计方法。根据 C 程序，我们知道，在这里我们需要：

1. 维护一个锁
2. 维护一段环形缓冲区

那么，数据结构可以设计为：

```rust
struct UartPointer(usize);

struct UartBuffer {
    buf: [u8; UART_TX_BUF_SIZE],
    uart_writer: UartPointer,
    uart_reader: UartPointer,
}

struct Uart {
    buffer: Spinlock<UartBuffer>,  // spinlock 是之前实现的，简单引入
    tx_r_chan: u8,
}
```

`UartPointer` 是一个 newtype：

```rust
#[derive(Copy, Clone, Eq, PartialEq)]
struct UartPointer(usize);

impl UartPointer {
    const fn new() -> Self {
        Self(0)
    }

    #[inline]
    fn wrapping_add(self, offset: usize) -> Self {
        Self((self.0 + offset) % UART_TX_BUF_SIZE)
    }

    #[inline]
    fn wrapping_sub(self, offset: usize) -> Self {
        Self((self.0 + UART_TX_BUF_SIZE - offset % UART_TX_BUF_SIZE) % UART_TX_BUF_SIZE)
    }
}
```

并实现 `From<UartPointer> for usize`、`Add<usize>`、`AddAssign<usize>`、`Sub<usize>`、`SubAssign<usize>`，来进一步封装环形缓冲区的操作。

环形缓冲区本身设计 `push` 和 `pop` 两个操作：

```rust
struct UartBuffer {
    buf: [u8; UART_TX_BUF_SIZE],
    uart_writer: UartPointer,
    uart_reader: UartPointer,
}

impl UartBuffer {
    fn new() -> Self {
        Self {
            buf: [0; UART_TX_BUF_SIZE],
            uart_writer: UartPointer::new(),
            uart_reader: UartPointer::new(),
        }
    }

    // 返回 Err：缓冲区已满
    // 返回 Ok：成功写入
    fn push(&mut self, value: u8) -> Result<(), u8> {
        if self.uart_writer == self.uart_reader - 1 {
            // 满了
            Err(value)
        } else {
            self.buf[usize::from(self.uart_writer)] = value;
            self.uart_writer += 1;
            Ok(())
        }
    }

    // 返回 None：缓冲区已空
    // 返回 Some：成功读取
    fn pop(&mut self) -> Option<u8> {
        if self.uart_writer == self.uart_reader {
            // 空
            None
        } else {
            let value = self.buf[usize::from(self.uart_reader)];
            self.uart_reader += 1;
            Some(value)
        }
    }
}
```

接下来就是 Uart 本身的设计了。

## Init

我们来实现 `uartinit` 函数。内容基本照抄 C 侧。

```rust
pub const fn new() -> Self {

pub fn init(&self) {
    unsafe { write_reg(IER, 0x00) };
    unsafe { write_reg(LCR, LCR_BAUD_LATCH) };
    unsafe { write_reg(0, 0x03) };
    unsafe { write_reg(1, 0x00) };
    unsafe { write_reg(LCR, LCR_EIGHT_BITS) };
    unsafe { write_reg(FCR, FCR_FIFO_ENABLE | FCR_FIFO_CLEAR) };
    unsafe { write_reg(IER, IER_RX_ENABLE | IER_TX_ENABLE) };
}
```

## uartputc

这里用到了 C 侧的一个全局变量，我们将其引入 `bindgen_wrapper.h`。

```c
extern volatile int panicked; // from printf.c
```

FFI 后 `volatile` 的信息就没了，所以我们还需要：

```rust
#[inline]
fn is_panicked() -> bool {
    unsafe { addr_of!(panicked).read_volatile() != 0 }
}
```

实现过程中发现了一个坑：C 侧如果 panic 了，就死循环。

我本来想用 `panic!`，但是多问了一句为啥 C 侧不用（C 也有 `panic()` 函数）。

原来是 panic 的默认处理会调用 `printf`，进而调用到 `uart`，所以在 `uart` 本身的错误处理不能这样。

```rust
if is_panicked() {
    loop {
        // 这里不应该使用 panic!
        // 因为 panic! 本身会调用 printf，进而一路调用回来...
        // 我们只能原地死循环
    }
}
```

然后就是下面的实现，这里同时体现出了 Rust 的好处和坏处：

```rust
loop {
    {
        let mut buf = self.buffer.lock();
        if let Ok(_) = buf.push(c) {
            // C 代码的其他处理在 push 里封装好了
            // 锁会在离开 putc 后自动释放
            self.start();
            break;
        } else {
            // 理论上我们这里需要 sleep，然后等待 start 函数来唤醒我们
            // 诶，问题来了... sleep 需要 CSpinLock（因为是 C 侧在进行操作）
            // 也就是说我们的 RAII 就失效了
            // 如果不传入锁，就会出现在 sleep 之前、锁释放之后的这一瞬间刚好被中断打断
            // 中断处理中尝试唤醒我们...但是我们还没睡
            // 就睡死了...对，死了
            // 所以我们要自己实现 rust_sleep
        }
    }
}
```

**好处**：

1. RAII 自动处理了锁，我们不需要手动释放，出了作用域就自动释放了
2. 我们的 `push` 和 `UartPointer` 进行了两重封装，这里非常简单

**坏处**：我们没办法用 C 版的 `sleep` 了，它是手动管理锁的。

所以我们要实现 `rust_sleep`。就把 `proctable.rs` 改名叫 `proc.rs`，然后塞进去吧。

```rust
pub fn rust_sleep<T: ?Sized, U>(chan: &T, lock: SpinLockGuard<'_, U>) {
    let p = unsafe { myproc() };

    unsafe { acquire(&mut ((*p).lock)) };
    drop(lock); // 释放锁
    unsafe {
        (*p).chan = chan as *const T as *mut c_void;
        (*p).state = procstate::SLEEPING;
        sched();
        (*p).chan = core::ptr::null_mut();
        release(&mut ((*p).lock));
    }
}
```

顺便做了对 C 版本的封装：

```rust
/// 对 xv6 `sleep(chan, lk)` 的薄封装。
/// - 当前上下文允许睡眠，且调用前已经持有 `lock`。
pub fn sleep<T: ?Sized>(chan: &T, lock: &CSpinLock) {
    unsafe {
        c_sleep(
            chan as *const T as *const c_void as *mut c_void as *const CSpinLock
                as *const spinlock as *mut spinlock,
        )
    };
    // 需要进行一系列转换
    // 这一连串转换好搞笑
}

/// 对 xv6 `wakeup(chan)` 的薄封装。
pub fn wakeup<T: ?Sized>(chan: &T) {
    unsafe { c_wakeup(chan as *const T as *const c_void as *mut c_void) };
}
```

这里我们可以和 C 版本对照一下，很有意思。

```c
sleep(void *chan, struct spinlock *lk)
{
    struct proc *p = myproc();

    // Must acquire p->lock in order to
    // change p->state and then call sched.
    // Once we hold p->lock, we can be
    // guaranteed that we won't miss any wakeup
    // (wakeup locks p->lock),
    // so it's okay to release lk.

    acquire(&p->lock);  // DOC: sleeplock1
    release(lk);

    // Go to sleep.
    p->chan = chan;
    p->state = SLEEPING;

    sched();

    // Tidy up.
    p->chan = 0;

    // Reacquire original lock.
    release(&p->lock);
    acquire(lk);
}
```

这里重点看锁的处理。我们使用 `drop + RAII` 释放锁，这没问题。

问题在于，我们没有办法 acquire 锁，因为我们的 Guard 已经被释放了！

我是怎么处理的呢？将这个函数的行为定义为"需要用户额外获得锁"。

这其实相当符合 Rust 的所有权机制。你可以注意到，我们的 `lock` 是直接获得的所有权，而不是借用。

因此，这个函数调用完毕后，外部就没有 `lock` 的所有权了。程序员想用 `lock`（来访问数据），就会发现 `lock` 没了，自然会去拿一个锁。

而很显然，这个地方的锁早一点晚一点，不会像开头那么影响。

这里还有一个借用检查的小坑：如果直接把 `&buf.uart_writer` 或 `&buf.uart_reader` 传给 `rust_sleep`，同时再把 `buf` 这个 guard move 进去，Rust 会拒绝，因为你一边借用了 `buf` 的内部字段，一边又要销毁整个 guard。

所以这里我给 `Uart` 单独加了一个 `tx_r_chan: u8`，作为睡眠/唤醒用的 channel。

于是乎，我们只需要在 `else` 里调用：

```rust
rust_sleep(&self.tx_r_chan, buf);
```

而 `start()` 里对应写：

```rust
wakeup(&self.tx_r_chan);
```

即可。一轮 loop 结束后，自然会再拿一个锁。

## uartstart

这里出现了一层张力……C 通过注释要求调用 `start` 之前必须持锁，而我们的 `start` 显然需要一个锁进行操作（因为我们是 RAII）。

所以，我们这里的 `start` 和 C 不一样，需要从外部借用一个锁。

```rust
fn start(&self, buf: &mut SpinLockGuard<'_, UartBuffer>) {
    // 这里存在一层问题... C 里面是"必须外部持锁"
    // 这里我们通过从外部借一个锁实现
    loop {
        match buf.pop() {
            Some(c) if (unsafe { read_reg(LSR) } & LSR_TX_IDLE) == 1 => {
                wakeup(&self.tx_r_chan);
                unsafe {
                    write_reg(THR, c);
                }
            }
            _ => return,
        }
    }
}
```

同样地，上面调用 `start` 的地方也需要传入一个可变借用。

## 微重构：read_reg/write_reg

这两个函数现在只是 `xx_volatile` 的封装，而传入参数是等效于整数的。也就是说，我们要求外部调用者必须确保传入参数是合法的值，因此这里是 `unsafe` 的函数。

如果我们将 `RegOffset` 和 `RegValue` 变成 newtype，并限制其可能的值，那么我们就无法表达出错误状态。

```rust
enum RegFlag {
    IerRxEnable,
    IerTxEnable,
    FcrFifoEnable,
    FcrFifoClear,
    LcrEightBits,
    LcrBaudLatch,
    LsrTxIdle,
}

impl RegValue {
    const ZERO: Self = Self(0);

    #[inline]
    const fn new(flag: RegFlag) -> Self {
        Self(match flag {
            RegFlag::IerRxEnable => 1 << 0,
            RegFlag::IerTxEnable => 1 << 1,
            RegFlag::FcrFifoEnable => 1 << 0,
            RegFlag::FcrFifoClear => 3 << 1,
            RegFlag::LcrEightBits => 3 << 0,
            RegFlag::LcrBaudLatch => 1 << 7,
            RegFlag::LsrTxIdle => 1 << 5,
        })
    }

    #[inline]
    const fn from_raw(raw: u8) -> Self {
        Self(raw)
    }

    #[inline]
    const fn raw(self) -> u8 {
        self.0
    }

    #[inline]
    const fn contains(self, flag: RegFlag) -> bool {
        (self.0 & Self::new(flag).0) == Self::new(flag).0
    }
}
```

同时，我们也给 `RegValue` 实现了加减，等效于底层的或/与运算。我们也为 `RegValue` 和 `RegFlag` 实现了 `PartialEq`，这样我们可以简单地使用 `==`、`!=` 来处理逻辑。

以及 `Reg` 和配套函数：

```rust
enum Reg {
    Thr,
    Ier,
    Fcr,
    Lcr,
    Lsr,
    Dll,
    Dlm,
}

const fn reg_offset(reg: Reg) -> usize {
    match reg {
        Reg::Thr | Reg::Dll | Reg::Rhr => 0,
        Reg::Ier | Reg::Dlm => 1,
        Reg::Fcr => 2,
        Reg::Lcr => 3,
        Reg::Lsr => 5,
    }
}

#[inline]
fn read_reg(reg_name: Reg) -> RegValue {
    RegValue::from_raw(unsafe {
        core::ptr::read_volatile(reg(reg_offset(reg_name)).cast_const())
    })
}

#[inline]
fn write_reg(reg_name: Reg, value: RegValue) {
    unsafe { core::ptr::write_volatile(reg(reg_offset(reg_name)), value.raw()) };
}
```

这样，我们的初始化就可以改成：

```rust
write_reg(Reg::Ier, RegValue::ZERO);
write_reg(Reg::Lcr, RegValue::new(RegFlag::LcrBaudLatch));
write_reg(
    Reg::Dll,
    RegValue::new(RegFlag::IerRxEnable) + RegFlag::IerTxEnable,
);
write_reg(Reg::Dlm, RegValue::ZERO);
write_reg(Reg::Lcr, RegValue::new(RegFlag::LcrEightBits));
write_reg(
    Reg::Fcr,
    RegValue::new(RegFlag::FcrFifoEnable) + RegFlag::FcrFifoClear,
);
write_reg(
    Reg::Ier,
    RegValue::new(RegFlag::IerRxEnable) + RegFlag::IerTxEnable,
);
```

## uartputc_sync

C 侧实现了绕过我们的一系列锁机制、直接忙等待的 `putc` 版本，我们也提供相同语义。

```rust
fn putc_sync(&self, c: u8) {
    // 关闭中断
    unsafe { push_off(); }
    if is_panicked() {
        loop {}
    }
    while read_reg(Reg::Lsr) != RegFlag::LsrTxIdle {
        // 等待
    }
    write_reg(Reg::Thr, RegValue::from_raw(c));
    unsafe { pop_off() };
}
```

这里由于我们实现了 `PartialEq`，因此可以使用 `!=` 比较 `RegFlag` 和 `RegValue`。

## uartgetc

由于抽象层都搭建好了，后面的函数都很简单：

```rust
fn getc() -> Option<u8> {
    if read_reg(Reg::Lsr) == RegFlag::LsrDataReady {
        Some(read_reg(Reg::Rhr).raw())
    } else {
        None
    }
}
```

这里需要注意，我们回过头去加了 `RegFlag::LsrDataReady` 和 `Reg::Rhr` 两个枚举，最初忘了写了。同时，我们使用 `Option` 返回结果。

## uartintr

```rust
fn intr(&self) {
    loop {
        match self.getc() {
            Some(c) => unsafe { consoleintr(c as i32); },
            None => break,
        }
```

### 然后不出意外地出意外了

```c
(ReadReg(LSR) & LSR_TX_IDLE) == 0
```

```rust
read_reg(Reg::Lsr).contains(RegFlag::LsrTxIdle) == false
```

当然，从"写进硬件的数值"来说，它其实没错，因为：

```rust
(1 << 0) | (1 << 1) = 0x03
```

我最初写成这样，也是因为当时没仔细看：看到 `0x03`，理所当然想到"`0x01 | 0x01<<1`"，回去查一下定义，拼出了这个结果。

但是语义上这就很不对劲了。`DLL=0x03` 表示的是波特率分频，`IER_RX_ENABLE | IER_TX_ENABLE` 表示的是中断使能位。这两个东西只是数值碰巧一样，不代表可以混着写。

于是，这里我最后还是单独补了一个：

```rust
RegFlag::DllBaud38K4
```

然后改成：

```rust
write_reg(Reg::Dll, RegValue::new(RegFlag::DllBaud38K4));
```

真正的问题在 `getc()`。

和前面一样，`==` 应该换成 `contains`。

```rust
fn getc(&self) -> Option<u8> {
    if read_reg(Reg::Lsr).contains(RegFlag::LsrDataReady) {
        Some(read_reg(Reg::Rhr).raw())
    } else {
        None
    }
}
```

最终代码：

```rust
use crate::bindings::{consoleintr, panicked, pop_off, push_off};
use crate::proc::{rust_sleep, wakeup};
use core::ptr::addr_of;
use spinlock::{SpinLock, SpinLockGuard};

use core::ops::{Add, AddAssign, Sub, SubAssign};

#[derive(Copy, Clone, Eq, PartialEq)]
struct RegValue(u8);

#[derive(Copy, Clone, Eq, PartialEq)]
enum RegFlag {
    IerRxEnable,
    IerTxEnable,
    DllBaud38K4,
    FcrFifoEnable,
    FcrFifoClear,
    LcrEightBits,
    LcrBaudLatch,
    LsrTxIdle,
    LsrDataReady,
}

impl RegValue {
    const ZERO: Self = Self(0);

    #[inline]
    const fn new(flag: RegFlag) -> Self {
        Self(match flag {
            RegFlag::IerRxEnable => 1 << 0,
            RegFlag::IerTxEnable => 1 << 1,
            RegFlag::DllBaud38K4 => 0x03,
            RegFlag::FcrFifoEnable => 1 << 0,
            RegFlag::FcrFifoClear => 3 << 1,
            RegFlag::LcrEightBits => 3 << 0,
            RegFlag::LcrBaudLatch => 1 << 7,
            RegFlag::LsrTxIdle => 1 << 5,
            RegFlag::LsrDataReady => 1 << 0,
        })
    }

    #[inline]
    const fn from_raw(raw: u8) -> Self {
        Self(raw)
    }

    #[inline]
    const fn raw(self) -> u8 {
        self.0
    }

    #[inline]
    const fn contains(self, flag: RegFlag) -> bool {
        (self.0 & Self::new(flag).0) == Self::new(flag).0
    }
}

impl Add<RegFlag> for RegValue {
    type Output = Self;

    fn add(self, rhs: RegFlag) -> Self::Output {
        Self(self.0 | Self::new(rhs).0)
    }
}

impl AddAssign<RegFlag> for RegValue {
    fn add_assign(&mut self, rhs: RegFlag) {
        self.0 |= Self::new(rhs).0;
    }
}

impl Sub<RegFlag> for RegValue {
    type Output = Self;

    fn sub(self, rhs: RegFlag) -> Self::Output {
        Self(self.0 & !Self::new(rhs).0)
    }
}

impl SubAssign<RegFlag> for RegValue {
    fn sub_assign(&mut self, rhs: RegFlag) {
        self.0 &= !Self::new(rhs).0;
    }
}

impl PartialEq<RegFlag> for RegValue {
    fn eq(&self, other: &RegFlag) -> bool {
        self.0 == Self::new(*other).0
    }
}

impl PartialEq<RegValue> for RegFlag {
    fn eq(&self, other: &RegValue) -> bool {
        RegValue::new(*self).0 == other.0
    }
}

#[derive(Copy, Clone)]
enum Reg {
    Thr,
    Ier,
    Fcr,
    Lcr,
    Lsr,
    Dll,
    Dlm,
    Rhr,
}

pub const UART0: usize = 0x1000_0000;
pub const UART_TX_BUF_SIZE: usize = 32;

static UART: Uart = Uart::new();

#[inline]
const fn reg(offset: usize) -> *mut u8 {
    (UART0 + offset) as *mut u8
}

const fn reg_offset(reg: Reg) -> usize {
    match reg {
        Reg::Thr | Reg::Dll | Reg::Rhr => 0,
        Reg::Ier | Reg::Dlm => 1,
        Reg::Fcr => 2,
        Reg::Lcr => 3,
        Reg::Lsr => 5,
    }
}

#[inline]
fn read_reg(reg_name: Reg) -> RegValue {
    RegValue::from_raw(unsafe {
        core::ptr::read_volatile(reg(reg_offset(reg_name)).cast_const())
    })
}

#[inline]
fn write_reg(reg_name: Reg, value: RegValue) {
    unsafe { core::ptr::write_volatile(reg(reg_offset(reg_name)), value.raw()) };
}

#[inline]
fn is_panicked() -> bool {
    unsafe { addr_of!(panicked).read_volatile() != 0 }
}

#[derive(Copy, Clone, Eq, PartialEq)]
struct UartPointer(usize);

impl UartPointer {
    const fn new() -> Self {
        Self(0)
    }

    #[inline]
    fn wrapping_add(self, offset: usize) -> Self {
        Self((self.0 + offset) % UART_TX_BUF_SIZE)
    }

    #[inline]
    fn wrapping_sub(self, offset: usize) -> Self {
        Self((self.0 + UART_TX_BUF_SIZE - offset % UART_TX_BUF_SIZE) % UART_TX_BUF_SIZE)
    }
}

impl From<UartPointer> for usize {
    fn from(value: UartPointer) -> Self {
        value.0
    }
}

impl core::ops::Add<usize> for UartPointer {
    type Output = Self;

    fn add(self, rhs: usize) -> Self::Output {
        self.wrapping_add(rhs)
    }
}

impl core::ops::AddAssign<usize> for UartPointer {
    fn add_assign(&mut self, rhs: usize) {
        *self = self.wrapping_add(rhs);
    }
}

impl core::ops::Sub<usize> for UartPointer {
    type Output = Self;

    fn sub(self, rhs: usize) -> Self::Output {
        self.wrapping_sub(rhs)
    }
}

impl core::ops::SubAssign<usize> for UartPointer {
    fn sub_assign(&mut self, rhs: usize) {
        *self = self.wrapping_sub(rhs);
    }
}

struct UartBuffer {
    buf: [u8; UART_TX_BUF_SIZE],
    uart_writer: UartPointer,
    uart_reader: UartPointer,
}

impl UartBuffer {
    const fn new() -> Self {
        Self {
            buf: [0; UART_TX_BUF_SIZE],
            uart_writer: UartPointer::new(),
            uart_reader: UartPointer::new(),
        }
    }

    fn push(&mut self, value: u8) -> Result<(), u8> {
        if self.uart_writer == self.uart_reader - 1 {
            Err(value)
        } else {
            self.buf[usize::from(self.uart_writer)] = value;
            self.uart_writer += 1;
            Ok(())
        }
    }

    fn pop(&mut self) -> Option<u8> {
        if self.uart_writer == self.uart_reader {
            None
        } else {
            let value = self.buf[usize::from(self.uart_reader)];
            self.uart_reader += 1;
            Some(value)
        }
    }
}

pub struct Uart {
    buffer: SpinLock<UartBuffer>,
    tx_r_chan: u8,
}

impl Uart {
    pub const fn new() -> Self {
        Self {
            buffer: SpinLock::new(UartBuffer::new(), "uart"),
            tx_r_chan: 0,
        }
    }

    pub fn init(&self) {
        write_reg(Reg::Ier, RegValue::ZERO);
        write_reg(Reg::Lcr, RegValue::new(RegFlag::LcrBaudLatch));
        write_reg(Reg::Dll, RegValue::new(RegFlag::DllBaud38K4));
        write_reg(Reg::Dlm, RegValue::ZERO);
        write_reg(Reg::Lcr, RegValue::new(RegFlag::LcrEightBits));
        write_reg(
            Reg::Fcr,
            RegValue::new(RegFlag::FcrFifoEnable) + RegFlag::FcrFifoClear,
        );
        write_reg(
            Reg::Ier,
            RegValue::new(RegFlag::IerRxEnable) + RegFlag::IerTxEnable,
        );
    }

    pub fn putc(&self, c: u8) {
        if is_panicked() {
            loop {}
        }

        loop {
            {
                let mut buf = self.buffer.lock();
                if let Ok(()) = buf.push(c) {
                    self.start(&mut buf);
                    break;
                } else {
                    rust_sleep(&self.tx_r_chan, buf);
                }
            }
        }
    }

    fn start(&self, buf: &mut SpinLockGuard<'_, UartBuffer>) {
        // 这里存在一层问题... C 里面是"必须外部持锁"
        // 这里我们通过从外部借一个锁实现
        loop {
            match buf.pop() {
                Some(c) if read_reg(Reg::Lsr).contains(RegFlag::LsrTxIdle) => {
                    wakeup(&self.tx_r_chan);
                    write_reg(Reg::Thr, RegValue::from_raw(c));
                }
                _ => return,
            }
        }
    }

    fn putc_sync(&self, c: u8) {
        // 关闭中断
        unsafe {
            push_off();
        }
        if is_panicked() {
            loop {}
        }
        while read_reg(Reg::Lsr).contains(RegFlag::LsrTxIdle) == false {
            // 等待
        }
        write_reg(Reg::Thr, RegValue::from_raw(c));
        unsafe { pop_off() };
    }

    fn getc(&self) -> Option<u8> {
        if read_reg(Reg::Lsr).contains(RegFlag::LsrDataReady) {
            Some(read_reg(Reg::Rhr).raw())
        } else {
            None
        }
    }

    fn intr(&self) {
        loop {
            match self.getc() {
                Some(c) => unsafe {
                    consoleintr(c as i32);
                },
                None => break,
            }
        }
        let mut lock = self.buffer.lock();
        self.start(&mut lock);
    }
}

#[unsafe(no_mangle)]
pub extern "C" fn uartinit() {
    UART.init();
}

#[unsafe(no_mangle)]
pub extern "C" fn uartintr() {
    UART.intr();
}

#[unsafe(no_mangle)]
pub extern "C" fn uartputc(c: i32) {
    UART.putc(c as u8);
}

#[unsafe(no_mangle)]
pub extern "C" fn uartputc_sync(c: i32) {
    UART.putc_sync(c as u8);
}

#[unsafe(no_mangle)]
pub extern "C" fn uartgetc() -> i32 {
    match UART.getc() {
        Some(c) => c as i32,
        None => -1,
    }
}
```
